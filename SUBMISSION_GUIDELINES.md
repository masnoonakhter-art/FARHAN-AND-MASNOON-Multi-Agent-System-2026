# Submission Guidelines

## How to Submit Your Work

### 1. Folder Structure
Each team member should organize submissions as follows:

```
submissions/
├── Farhan_Masnoon/
│   ├── marketing/
│   │   ├── market_research.md
│   │   ├── marketing_strategy.doc
│   │   ├── content_calendar.xlsx
│   │   └── campaign_plan.pdf
│   ├── development/
│   │   ├── code/
│   │   ├── tests/
│   │   ├── documentation/
│   │   └── api_docs.md
│   └── research/
│       ├── research_reports/
│       ├── technology_analysis.md
│       └── literature_review.md
│
└── Masnoon_Akhter/
    ├── marketing/
    ├── development/
    └── research/
```

### 2. File Naming Convention
- Use descriptive names with underscores: `market_research_2026.md`
- Include dates when applicable: `report_june_2026.pdf`
- Use clear, professional naming

### 3. Documentation Requirements
Each submission should include:
- **Title**: Clear indication of what the work is
- **Author**: Your name
- **Date**: Submission date
- **Summary**: Brief overview of contents
- **Status**: Completed/In Progress/Pending Review
- **References**: Any sources or citations used

### 4. Submission Checklist
Before submitting, ensure:
- [ ] All files are clearly named and organized
- [ ] Documentation is complete and professional
- [ ] Code follows PEP 8 standards (if applicable)
- [ ] All deliverables are included
- [ ] Work is saved with proper formatting
- [ ] README or overview file is included in each folder

### 5. Pull Request Process
When submitting code or major updates:
1. Create a branch: `feature/farhan-marketing` or `feature/masnoon-development`
2. Commit your work with clear messages
3. Create a Pull Request with description
4. Request review from project coordinator
5. Address feedback and merge

### 6. Timeline
- **Phase 1 (Marketing)**: Complete by mid-project
- **Phase 2 (Development)**: Complete by 3/4 project mark
- **Phase 3 (Research)**: Ongoing, finalize by project end

### 7. Communication
- Update progress in comments
- Use GitHub Issues for blockers or questions
- Tag project coordinator in important updates
- Regular check-ins recommended
